# GIT

## Merge

- 합치는 과정 히스토리에 남아있음

## Rebase

- 히스토리를 남기지 않고 머지
- 기록이 사라짐 => 장점은 깔끔, 단점은 기록 삭제

## HEAD

- 브랜치에는 숨겨진 녀석이 있다.

- 현재 작업중인 커밋

- ```
  // 하나 이전의 커밋으로 돌아감
  git switch HEAD^
  // 두개 이전으로 돌아감
  git switch HEAD^^
  // 틸드를 사용한 상대참조
  git branch -f step2 step 1~2
  ```

## 되돌리기

- 완전히 기록을 지우는 방법 (로컬에서 작업 시 추천)

  - ```
    // 최근 커밋 1개 제거
    git reset HEAD~1
    ```

- 다른 사람의 리모트 브렌치에서 작업중인경우

  - ```
    // 다시 커밋을 통해 다른 사용자에게 변경내역을 알려줌
    git revert HEAD
    ```



## 삭제된 커밋, 브랜치 복구

- 해시로 저장이 되기 떄문에 해당 해시값을 찾아서 리셋시킴

- 커밋 되돌리기

  - ```
    // 모든 로그를 볼 수 있음 (해시값포함)
    git reflog
    // git reset --hard ${hash}
    ```

- 삭제한 브랜치 복구하는 법

  - ```
    git switch -c step3 ${hash}
    ```
